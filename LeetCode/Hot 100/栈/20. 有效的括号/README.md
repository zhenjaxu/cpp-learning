# 20.有效的括号
## 理解
判断字符串的括号是否有效
### 理解一，遍历
将遍历过的括号放入栈中。除了第一次，每次遍历都判断栈顶是否与当前遍历括号是否配对。
配对成功则弹出，没有配对说明括号无效。
如果是左括号，就存入栈内；如果是右括号，就与栈顶的左括号进行配对。
### 理解二，指针
理解一的空间复杂度为O（N），那有没有O（1）的方法解决该问题。
一般通过几个指针来实现O（1）。
两个指针，一前一后，分别判断括号的左右两边。这样下来，需要多次遍历整个字符串才能做到。
两个指针分别从首尾出发，观察指针所指是否配对。
刚开始不能直接判断，如果括号都是一对一对的，那首位不一定是配对的。
所以尾指针需找到与首指针配对位置。那就不应该从尾部出发，而因从头部出发。如果后面也有一对相同的括号就会出问题。
但如果从头部出发也会出现同样的问题。两个指针实现过于困难。
重新思考一下，有效括号是以一对或多对有效括号组成的。难点就在于如何遍历的反括号是否是当前这对的。
而且这个方法还需要借助递归，所以O（1）空间复杂度难以实现。
大概步骤是先分对，然后判断每一对内是否同样有效。这样一来就需要每一对调用一次该函数。
如何判断一对一对的，通过判断前一对的右括号后一对的左括号相邻。
可是一对里面也可能出现这种情况。怎么想都达不到那种效果。故而放弃，看看题解吧。

## 尝试写
### 函数声明
```cpp
bool isValid(string s);
```
### 方法一，遍历
尝试理解一，利用栈存放遍历过的左括号。
```cpp
std::stack<char> leftBrackets;
for(size_t i=0;i<s.size();++i){
    if((s[i]==')'||s[i]==']'||s[i]=='}')){
        if(leftBrackets.empty()) return false;
        else{
            char top=leftBrackets.top();
            if((s[i]==')'&&top!='(')||
                (s[i]==']'&&top!='[')||
                (s[i]=='}'&&top!='{')){
                return false;
            }
            leftBrackets.pop();
        }
    }
    else leftBrackets.push(s[i]);   
}
return leftBrackets.empty()?true:false;
```
时间复杂度O（N），空间复杂度O（N）

## 题解
### 栈
判断括号的有效性可以使用「栈」这一数据结构来解决。
我们遍历给定的字符串 s。当我们遇到一个左括号时，我们会期望在后续的遍历中，有一个相同类型的右括号将其闭合。
由于后遇到的左括号要先闭合，因此我们可以将这个左括号放入栈顶。
当我们遇到一个右括号时，我们需要将一个相同类型的左括号闭合。此时，我们可以取出栈顶的左括号并判断它们是否是相同类型的括号。
如果不是相同的类型，或者栈中并没有左括号，那么字符串 s 无效，返回 False。
为了快速判断括号的类型，我们可以使用哈希表存储每一种括号。哈希表的键为右括号，值为相同类型的左括号。
在遍历结束后，如果栈中没有左括号，说明我们将字符串 s 中的所有左括号闭合，返回 True，否则返回 False。
注意到有效字符串的长度一定为偶数，因此如果字符串的长度为奇数，我们可以直接返回 False，省去后续的遍历判断过程。
```cpp
int n = s.size();
if (n % 2 == 1) {
    return false;
}
unordered_map<char, char> pairs = {
    {')', '('},
    {']', '['},
    {'}', '{'}
};
stack<char> stk;
for (char ch: s) {
    if (pairs.count(ch)) {
        if (stk.empty() || stk.top() != pairs[ch]) {
            return false;
        }
        stk.pop();
    }
    else {
        stk.push(ch);
    }
}
return stk.empty();
```
时间复杂度：O(n)，其中 n 是字符串 s 的长度。
空间复杂度：O(n+∣Σ∣)，其中 Σ 表示字符集，本题中字符串只包含 6 种括号，∣Σ∣=6。栈中的字符数量为 O(n)，而哈希表使用的空间为 O(∣Σ∣)，相加即可得到总空间复杂度。
看了题解之后感觉还能对方法一进行修改，使之跟简洁。
用一个unorderedmap去存放括号，右括号作索引，左括号作值，方便直接与栈顶元素进行比较。
同时，也避免了重复写括号的繁琐。不过空间复杂度会加上unorderedmap的长度。
for循环通过迭代的写法直接遍历s中的元素，无需索引取值了。
通过判断字符串长度是否为偶数来做一个初步筛选，提高整体运行效率。
```cpp
if(s.size()%2==1) return false;
std::unordered_map<char, char> pairs{
    {')', '('},
    {']', '['},
    {'}', '{'}
};
std::stack<char> leftBrackets;
for(char ch:s){
    if(pairs.count(ch)){
        if(leftBrackets.empty()||
            pairs[ch]!=leftBrackets.top()) return false;
        leftBrackets.pop();
    }
    else leftBrackets.push(ch);   
}
return leftBrackets.empty()?true:false;
```
时间复杂度O（N），空间复杂度O（n+|e|）。
